◇ 001 — SYSTEM TYCHO An experimental systems language — transpiles to C

Automatic memory management. No garbage collector, no manual free.

A research project testing one idea: implicit hierarchical arenas under value semantics. Each scope owns a memory arena, freed on exit; with no reference type, the compiler places every allocation and free from your code's structure alone. Builds with a plain cc and make. 

fn greet(name: string) -> string:
    return "hello " + name

fn main():
    name := input()
    println(greet(name))

01 How it works

Memory comes from scope, not a runtime.

  • Each scope owns a memory arena — allocation is a pointer bump; the whole arena is freed the moment the scope exits.
  • There's no reference type. b := a copies, so the compiler sees every value's lifetime from the syntax alone.
  • So it inserts every allocation and free for you: no garbage collector, no manual free, nothing to annotate.

A string built across a loop lives in main's arena, freed for you on return

fn main():
    total := ""
    for i in range(1, 6):
        total = total + str(i)
        if i < 5:
            total = total + ", "
    println("counted: " + total)   # counted: 1, 2, 3, 4, 5

02 Why it stays fast

Value semantics, without the copies you'd expect.

  • A value can leave a scope only two ways, both visible in the source: down (passed into a callee) or up (returned to the caller).
  • A value you build and return is allocated straight into the caller's arena — return moves it, it doesn't copy.
  • acc = acc + x in a loop grows one buffer in place: the textbook O(n²) string build becomes O(n).

Both are sound because a value is never aliased — a copy is always independent

struct Person:
    name: string
    age: int

fn promote(p: Person) -> Person:
    q := p                # a full, independent copy
    q.age = p.age + 1
    return q             # moved into caller's arena, not copied

fn main():
    ada := Person("Ada", 36)
    older := promote(ada)
    println(str(ada.age) + " " + str(older.age))   # 36 37

03 The language

Small, and Go/Odin-shaped.

Python/Nim-flavored syntax, Go/Odin-like semantics; the value-semantics core comes from Hylo. What's in the box:

  • Value semantics — no aliasing, no GC
  • Generics — $T, monomorphized
  • Sum types — Option/Result, match
  • Methods (UFCS) — no classes
  • Closures — value-capture
  • Concurrency — spawn, channels, parallel-for
  • FFI to C — extern fn, handles, bytes
  • Packages + corelib — import, 36 modules

04 How it's checked

An experiment, but a heavily-checked one.

Tycho is a research project — an experiment testing one idea — but heavily checked, which is the part most experiments this young skip. It's experimental in scope, not in rigor.

self-hostTwo compilers: a reference written in C, and a second one written in Tycho that compiles itself, running on the same implicit arenas it emits. make fixpoint holds it to reproducing its own emitted C byte-for-byte, and to matching the C reference on every test.
fuzzA differential fuzzer runs both compilers on random programs under ASan/UBSan; every example is built twice — native and sanitized — and checked against a committed golden output.
localThere's no cloud CI to take on faith. make ci runs the whole gate on your own machine.

Where the model helps and where it costs are both measured — as machine-specific ratios, with the workloads that produce them — in the performance notes. Pointer-shaped data like graphs and trees is where value semantics is least free, and the docs are honest about that, not just about the wins.
